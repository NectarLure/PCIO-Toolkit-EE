# 专家填写说明

## 审核目标

本次专家审核用于评价 PCIO Toolkit 作为 DSR 人工制品的设计合理性、模块完整性、场景适配性和证据边界清晰性。

本次审核不要求专家判断工具是否已经带来绩效提升，因为当前材料不包含纵向复测或因果验证数据。

## 建议阅读顺序

请按以下顺序阅读材料包：

1. `research_brief.md`：了解研究定位、目标和证据边界。
2. `pcio_model_brief.md`：审阅 PCIO 四层模型。
3. `artifact_overview_brief.md`：审阅工具包模块与设计原则。
4. `case_p_summary.md`：审阅流程生产型企业 T0 基线诊断。
5. `case_t_summary.md`：审阅罐区储运运营场景 T0 基线诊断。
6. `evidence_boundary_table.md`：确认不同证据类型的可支持结论。
7. `expert_review_form.md`：填写评分和开放式意见。

## 如何评分

请使用 1—5 分 Likert 量表：

- 1 = 非常不同意 / 明显不足
- 2 = 不同意 / 存在较多不足
- 3 = 一般 / 基本可接受但需要修改
- 4 = 同意 / 较合理
- 5 = 非常同意 / 很合理

请尽量为每个评分项写一句简短理由。若无法判断，请说明需要补充的材料或证据。

## 如何提交反馈

可在 `expert_review_form.md` 中填写评分、理由和开放式问题答案，然后通过研究团队指定方式返回。若研究团队没有指定提交方式，请先确认提交渠道。

请勿将包含专家身份信息或未公开意见的文件上传到公开仓库。

## 隐私与脱敏要求

请勿在反馈中写入真实企业名称、员工姓名、手机号、微信、邮箱、客户、供应商、合同号、项目名、精确地址、真实设备编号、车牌或罐区安防细节。

如需举例，请使用：

- Case P；
- Case T；
- 设备M01；
- Area 1；
- Line 1；
- 通用岗位或通用流程名称。

## 证据边界提醒

- 真实案例仅为 T0 基线诊断案例。
- 合成案例仅用于流程测试和规则响应演示。
- 自动化测试仅支持技术正确性、功能完整性和模块运行一致性。
- ROI 模块仅为情景估算工具。
- 当前材料不支持部署后改善、因果效果、真实 ROI 实现或长期运行结论。
