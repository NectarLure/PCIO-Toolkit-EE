# PCIO Toolkit 工具包模块说明

## 工具包目标

PCIO Toolkit 是一个用于智能制造基线诊断和决策支持的 DSR 人工制品。其目标是把组织画像、题项诊断、风险识别、KPI 建议、ROI 情景和 PDCA 后续计划连接成可追溯流程。

## 模块结构

| 模块 | 功能定位 | 输出内容 | 证据边界 |
|---|---|---|---|
| 组织画像 | 界定组织场景、诊断范围和基础条件 | 场景摘要、诊断边界、适用说明 | 不构成绩效证据 |
| 诊断题项 | 收集智能制造相关基线信息 | 题项响应、PCIO 映射基础 | 小样本 T0 不支持量表效度结论 |
| PCIO 得分/映射 | 将题项映射到四层能力 | 分层基线、短板提示 | 支持基线诊断，不支持纵向变化 |
| 风险矩阵 | 根据短板和场景生成候选风险 | 风险清单、优先级、人工确认提示 | 风险需专家和现场人员确认 |
| KPI 建议 | 将诊断问题转化为可跟踪指标 | KPI 名称、口径、跟踪建议 | 不表示 KPI 已改善 |
| ROI 情景估算 | 支持方案比较和假设讨论 | 成本/收益假设、情景比较 | 不表示真实 ROI 实现 |
| 报告导出 | 形成结构化诊断报告 | 摘要、风险、KPI、PDCA 计划 | 报告是决策支持材料 |
| PDCA 复测 | 规划后续 T1/T2 复测 | 复测节点、责任人、检查项 | 复测计划不是复测结果 |

## 设计原则

| 编号 | 设计原则 | 含义 |
|---|---|---|
| DP1 | 基础能力优先，而非直接优化 | 先确认数据采集和连接基础，再讨论洞察和优化。 |
| DP2 | 证据关联的诊断与决策支持 | 风险和 KPI 建议应回连到题项、场景或规则依据。 |
| DP3 | 低成本、模块化部署 | 保持轻量、本地、可拆分的工具结构。 |
| DP4 | 人机协同与人工确认 | 所有风险、KPI 和 ROI 情景均需保留人工判断入口。 |
| DP5 | 基于 PDCA 的持续复测 | 工具生成复测计划，但不替代真实跟踪评价。 |
| DP6 | 情境适配与跨案例可比性 | 保留统一 PCIO 框架，同时允许不同场景差异化表达。 |

## 专家审核重点

请重点评价：

- 模块之间是否形成清楚的诊断链；
- 输出是否便于工程师、管理者或研究者理解；
- 风险和 KPI 是否足够可追溯；
- ROI 情景是否表达谨慎；
- PDCA 计划是否适合后续真实纵向评价。
