Evidence Types Are Not Interchangeable

Automated tests support module consistency. Synthetic cases demonstrate workflow and rule response. Real T0 cases support baseline diagnosis and contextual fit. None of these evidence types replaces longitudinal performance tracking or causal research design.

Automated Test Summary

Tests cover survey parsing, PCIO mapping, risk rules, KPI suggestions, ROI scenario estimation, report generation, export flow, and bilingual terminology consistency. This evidence supports whether modules run according to the design logic.

Technical correctness Functional completeness Module consistency No performance claim

Synthetic Case Testing

Synthetic data are used to check whether different inputs generate differentiated risks, KPIs, and reports. Synthetic cases are not field validation and cannot be used as evidence of real industrial effects.

Workflow demonstration Rule-response check Output differentiation Not real organisation data

Real T0 Baseline Cases

Case P and Case T are based on first-round baseline materials. The public site retains only anonymised sample counts, baseline patterns, PCIO gaps, candidate risks, KPI suggestions, and PDCA plans. Because no follow-up data are available, they must not be described as long-term operation results.

Baseline diagnosis Contextual fit Formative field evaluation Small sample

Evidence Still Needed

Future work should include expert review results, interviews, T1/T2 follow-up, objective KPI tracking, control or quasi-experimental designs, short-scale development, and psychometric testing. This public site contains only protocols and blank forms, not expert results.

Please distinguish technical feasibility, formative fit, and organisational performance evidence.
